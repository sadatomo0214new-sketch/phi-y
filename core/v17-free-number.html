0. Subject

In v0.1, Free Number was defined as:

自由数とは、結果から順序を産む数である。

In v0.2, this was connected to quaternions. v0.2.1 sharpens the constraints: result does not arbitrarily rewrite history; it selects a history that can close the loop.

Result does not rewrite anything arbitrarily.
It selects the history that can close the loop.

Japanese core: 結果は過去を勝手に書き換えるのではない。閉路を閉じられる履歴を選ぶ。

1. Core Thesis

四元数は順序で結果が変わる。
自由数は結果で順序が決まる。

The essence of quaternion rotation lies in non-commutativity:

q₁ q₂ ≠ q₂ q₁

If the order of rotation changes, the result changes. Therefore, order is not mere notation; it is operationally real.

Ordinary quaternion reading:

σ ⇒ E

Free Number inversion:

E ⇒ σ_E

Free Quaternion consistency loop:

E ⇒ σ_E ⇒ E′,  E′ ≈ E

2. Terms

2.1 Quaternion Rotation Sequence

Let the set of quaternion rotors be:

Q = {q₁, q₂, ..., q_n}

Let the set of possible operation orders be:

Σ_Q = {σ_i(Q)}

Because quaternion multiplication is non-commutative:

Eval_{σ_a}(Q) ≠ Eval_{σ_b}(Q)

Order difference can produce result difference.

2.2 Admissible Order Set

Free Number does not select from an unlimited fantasy space. It selects from an admissible order set.

Σ_Q^adm = { σ_i(Q) | σ_i is admissible under P, R, C }

where:

P = Pressure / generalized constraint field
R = Reference frame
C = consistency condition

This keeps inversion from becoming arbitrary retro-storytelling.

2.3 Result

Let the observed final attitude, phase, or state be:

E

This is not merely an output value. It is the observed fact of having arrived there.

2.4 Pressure

Let pressure be:

P

Pressure is the condition that bends which history is selected from the admissible history set.

Select(E, Σ_Q^adm; P₁) ≠ Select(E, Σ_Q^adm; P₂)

In this draft, P is not only physical pressure. It is a generalized constraint field that bends retro-selection.

Pressure may include:

constraint pressure
observational pressure
reference-frame pressure
semantic pressure
temporal pressure
consistency pressure

3. Definition of Free Quaternion

Free Quaternion 𝔔_E is the structure in which, from an observed result E, a non-commutative rotation order σ_E that establishes that result is retro-selected under pressure P, then tested by re-evaluation.

𝔔_E := ⟨E, Σ_Q^adm, P, σ_E⟩

where:

σ_E = Select(E, Σ_Q^adm; P)

4. Selection Criterion

The selected history is not arbitrary.

It is chosen by minimizing the mismatch between the re-evaluated result and the observed result under pressure.

σ_E = argmin_{σ ∈ Σ_Q^adm} D( Eval_σ(Q), E; P )

where D is a pressure-weighted mismatch function.

Thus, Free Number does not claim:

any result may choose any history

It claims:

result selects the admissible history that can close the loop under pressure-weighted consistency.

Japanese core: 選択される履歴は恣意的ではない。圧力Pによって重みづけられた不一致関数Dを最小化する履歴が σ_E として選ばれる。

5. Inversion

5.1 Definition

Inversion is the operation in a non-commutative system by which result retro-selects historical order.

Inv_P: E ↦ σ_E

or:

σ_E = Inv_P(E, Σ_Q^adm)

The important point is that inversion is not simple reverse playback.

It does not merely flip:

σ → E

into:

E → σ

Inversion means that result E, under pressure P, selects from admissible order set Σ_Q^adm the history σ_E that establishes itself.

5.2 Shortest Definition

反転とは、結果が履歴を選ぶことである。

More strictly:

反転とは、非可換演算系において、結果から履歴順序を選び直し、その履歴が再び結果を再現することで確定する閉路である。

6. Consistency Loop

The history selected from the result must be able to reproduce the result within tolerance.

E --Inv_P--> σ_E
σ_E --Eval--> E′

and:

d(E′, E) < ε

or, in short:

E′ ≈ E

Thus, the basic Free Quaternion loop is:

E ⇒ σ_E ⇒ E′,  d(E′, E) < ε

When this loop holds, the history is not mere interpretation. It is fixed as a consistent rotation chain that establishes the result.

7. Path Inversion

An ordinary Path extends from past toward result.

E = Eval(Path)

A Free-Number Path unfolds backward from result.

Path_E = Inv_P(E)

Thus:

Path_E := σ_E

Ordinary:

Path ⇒ E

Free Number:

E ⇒ Path_E

Path は過去ではなく、結果の影である。

Mathematically:

Path(E) := σ_E

12. Axioms v0.2.1

Axiom 1 — Reality of Non-Commutativity

In quaternion rotation, order difference has result difference.

q₁q₂ ≠ q₂q₁

Therefore, order is not mere notation. It is operationally real.

Axiom 2 — Non-Absolute Priority of Order

Operation order is not treated as the sole prior ground of result.

In a non-commutative system, σ ⇒ E holds. But Free Number also permits E ⇒ σ_E.

σ ⇒ E
alone does not close the system.

E ⇒ σ_E
must be added to form the consistency loop.

Thus, order is not fixed only on the cause side. It may be retro-selected from the result point under pressure.

Axiom 3 — Admissible Order Set

Result-driven selection occurs only over an admissible order set.

Σ_Q^adm = { σ_i(Q) | σ_i is admissible under P, R, C }

Axiom 4 — Result-Driven Selection

Result retro-selects operation order from the admissible order set.

σ_E = Select(E, Σ_Q^adm; P)

Axiom 5 — Selection Criterion

The selected order minimizes pressure-weighted mismatch.

σ_E = argmin_{σ ∈ Σ_Q^adm} D( Eval_σ(Q), E; P )

Axiom 6 — Pressure Mediation

Order selection is bent by pressure as a generalized constraint field.

Select(E, Σ_Q^adm; P₁) ≠ Select(E, Σ_Q^adm; P₂)

Axiom 7 — Consistency Loop

The selected history is fixed by reproducing the result through re-evaluation within tolerance.

E ⇒ σ_E ⇒ E′,  d(E′, E) < ε

Axiom 8 — Object Stability

In the Phi-Y interpretation layer, object is the locally stable node of the result-history-result consistency loop.

O_x = StableLoop(E_x, σ_{E_x}; P_x)

Axiom 9 — Time Propagation

In the Phi-Y temporal-reading layer, time is the propagation of consistency loops.
