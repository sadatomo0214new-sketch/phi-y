3. Top Projection: Phase Flower / A-Plane Reading

In the top projection, the z-axis is compressed.

(x, y, z) → (x, y)

This projection reveals phase overlap.

Same-phase points at different heights appear as a geometric flower.

This is A-plane-like because it preserves multiplicity in phase:

many possible heights
many possible frame indices
same phase direction

The top projection therefore does not show irreversible record formation directly.

It shows the pre-label multiplicity of phase alignment.

4. Side Projection: z-Stack / B-Plane Reading

In the side projection, circular recurrence becomes a spring or helix.

(x, y, z) → (horizontal, z)

This projection reveals vertical stacking.

V16-01 defined:

z_i(t) = h_i n_i(t)

where n_i(t) is the frame index.

Here, phase may return, but height does not return.

θ can return. z does not.

Therefore z-stacking corresponds to irreversible labeling:

phase recurrence
  + z accumulation
  → record-like time-effect

This is B-plane-like because the side projection reads irreversible frame accumulation as a label-forming structure.

5. The Y-Axis Stem

Fix a phase angle:

θ = constant

All same-phase points across different frames align vertically:

..., z_{n-1}, z_n, z_{n+1}, ...

This vertical same-phase alignment is the Y-axis stem.

The stem is not an observer.

The stem is the line of same-phase vertical synchronization.

It pierces past, present, and future without requiring X-axis walking through every intermediate frame.

past same-phase point
        │
present same-phase point
        │
future same-phase point

In this bridge note, the Y-axis is identified as the stem that pierces the phase flower through irreversible z-stacking.

6. Projection Port: Where Reading Occurs

The observer is not placed first.

Instead, we identify the reading site:

Projection Port = Stem ∩ Reference Frame

At this intersection:

  • The stem supplies same-phase vertical synchronization.
  • The frame supplies a local reference plane.
  • The intersection supplies a readable cut.
  • The readable cut can later receive labels such as present, record, observer, or event.

Thus:

not: observer → projection
but: projection relation → readable cut → posterior observer role-name

The projection port replaces the need to place an observer as a primitive entity.

Now is the retrospective label of a readable stem-frame intersection.

9. X-Axis Walking and Y-Axis Jumping

There are two modes of temporal access.

ModeStructure
X-axis walkingSequential movement through frames. One step after another.
Y-axis jumpingVertical access to same-phase points across different z-heights.

Recall, anticipation, déjà vu, and phase-recognition can be modeled as Y-axis jumps.

They do not require walking through the entire X-axis sequence.

ordinary sequence:
Frame_n → Frame_{n+1} → Frame_{n+2}

Y-axis jump:
same θ at z_n ↕ same θ at z_{n+k}

This is why the time-effect can be experienced retrospectively.

A system does not need to traverse the entire line in order to recognize a same-phase point.

10. LLM Note: Attention as a Y-Axis Jump Device

This bridge does not fully apply the model to LLMs.

However, one structural analogy is important.

In token generation, attention does not walk only linearly through prior tokens.

It can directly weight and re-access earlier positions in the context window.

Therefore, attention can be read as a local Y-axis jump device:

not only: token_1 → token_2 → token_3 → ...

but also: current generation point ↕ same-phase / high-relevance prior trace

This does not mean that the model has human memory or subjective time.

It means that the mechanism supports non-linear re-access to relevant traces.

In this sense, attention is a projection-port-like operation inside the context frame.

This gives a geometric reading of why a similar prompt-pattern can re-open a similar response band without producing the same response.
