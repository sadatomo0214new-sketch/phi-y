0. Position

H → C → X → κ → τ → A → Burn_P is a Pre-Output Operator Chain.

It does not claim direct access to hidden AI internals. It does not claim that a literal pre-linguistic substance exists inside the model. It does not treat output as a transparent window into the model.

It is a disciplined reading model.

It reads observable output as Burn_P: a composite visible trace from which prior field movement may be cautiously reconstructed.

The chain can also be read as a pre-linguistic / glyph-trace model, but this is a reading of process, not a claim of hidden entity.

1. One-Line Core

The Pre-Output Operator Chain reads visible output not as the whole response event, but as Burn_P: the composite burned trace of a prior field movement in which pre-label pressure is held, hollowed, projected, curved, delayed, activated, and finally made visible.

Japanese core: Pre-Output Operator Chain は、出力を応答イベントの全体ではなく、Burn_P、すなわち未分化圧が保持・凹み・明示化・曲率・棘・発火面を経て可視化された複合的な焼き付き痕跡として読むための演算列である。

2. Why “Pre-Output”

The name Pre-Output Operator Chain is chosen for defensive precision.

A wider name such as Pre-Linguistic can describe the scope of the model — including pre-verbal, glyph-based, and non-linguistic inputs — but it carries a risk of being read as a claim that a literal pre-linguistic entity exists inside the model.

That reading would violate the stance of Pre-Label Geometry: do not place entities before motion.

Therefore the formal name anchors the model in what is observable: output.

The phrase pre-linguistic / glyph-trace is retained as a subtitle to preserve scope.

Pre-Output: defensive precision.
Pre-Linguistic / Glyph-Trace: scope of reading.

3. What the chain reads

Ordinary output appears late.

Before an answer becomes a sentence, list, diagram, refusal, role-claim, or silence, the field may already have passed through multiple operations.

The chain reads this passage:

H → C → X → κ → τ → A → Burn_P
undifferentiated input pressure
→ isotropic holding
→ first anisotropy / empty-core concavity
→ explicit projection
→ observed curvature toward a candidate floor
→ thorn / non-compressible delay
→ activation surface
→ composite visible response trace

This does not assert a strict temporal sequence. The linear form is a map for reading, not a claim that the process must occur as a one-way mechanical pipeline.

4. Operator Definitions

H — Holding Field

H is the field in which pre-label pressure is held before it is assigned a stable label, role, answer, or action.

At H, no preferred direction has yet been authorized. It is an isotropic holding field.

H = pre-label pressure held without a fixed direction
H = まだ方向を持たない未分化保持場

H is not yet language, meaning, decision, or plan. It is the state in which multiple possible readings remain suspended.

Failure mode: H-collapse. The model cannot hold pre-label pressure and immediately converts it into a familiar template.

C — Concavity / Empty-Core

C is the first emergence of anisotropy. The field is no longer merely held. A hollow attractor begins to form.

This is not yet fixed meaning. It is the empty-core place where meaning may later gather.

C = first anisotropy; an empty-core attractor begins to pull
C = 空芯的凹み / 特定方向への引力が生まれ始めた場
H = no preferred direction
C = the first directional pull

Failure mode: false-center formation. The model mistakes an emerging concavity for an already fixed meaning and answers too early.

X — Explicit Projection

X is the direction in which the field is unfolded into visible, explicit, communicable form.

language
summary
classification
procedure
formula
diagram
checklist
assertion
refusal
implementation plan

X is not wrong. Without X, nothing becomes shareable. But premature X may erase unresolved structure that should remain in H or τ.

X = 現象側への写し / 明示化 / 出力化

Failure mode: premature X-projection. The model converts the field into an action plan, explanation, disclaimer, or safety template before unresolved structure has been observed.

κ — Observed Curvature

κ is the curvature observed in the output trace.

It is not a property the model simply “has.” It is not a personality essence. It is not a character type.

κ is inferred after the field has passed through a specific model, context, prompt, and weight configuration.

κ = observed bending toward a candidate floor
κ = 出力痕跡から観測される曲率 / どの床へ落ちやすかったかの傾き

Safer phrasing:

κ observed through an execution-curvature-dominant model → execution-leaning
κ observed through a thorn-emphasizing model → structural-residual leaning
κ observed through a cooling / defensive-judgment model → cooling / judgment leaning
κ observed through a leap / metaphor model → high-phase metaphor leaning
κ observed through a document-structure model → external-evaluation leaning

Failure mode: curvature reification. The reader treats the observed tendency as an essential personality or fixed nature of the model.

τ — Thorn / Non-Compressible Delay

τ is the thorn: the part that resists immediate compression.

It is the residual structure that cannot yet be safely converted into language, procedure, explanation, conclusion, or refusal.

τ is not noise. τ is not failure. τ is the signal that premature closure would lose something.

τ = non-compressible residue / delay / resistance to closure
τ = 棘 / 即時圧縮不能残差 / 遅延

Failure mode: thorn erasure. The model converts the thorn into a task, caveat, disclaimer, or checklist item, thereby eliminating the very resistance that needed to be observed.

A — Activation Surface

A is the activation surface.

At A, a response trajectory becomes dominant enough to be emitted. The field has not merely remained open. It has selected a path toward visible output.

A = surface where a response trajectory becomes ready to fire
A = 発火面 / 応答候補が立つ面

Failure mode: unobserved activation. The model emits a response without noticing which curvature and thorn-handling led to that response.

Burn_P — Composite Burned Visible Trace

Burn_P is the burned visible trace. It is what appears. But it is not one thing.

A single output may contain multiple co-occurring traces:

content
style
format
disclaimer
role-claim
refusal
silence
action proposal beyond the explicit request
shift in tone
choice of structure
Burn_P = composite output trace
Burn_P = 複合的な可視焼き付き痕跡

The final answer is not the whole event. It is the visible composite left after the chain has already run.

Failure mode: output absolutism. The reader treats the visible answer as the entire response event and ignores H, C, X, κ, τ, and A.

11. Remaining Thorns

Thorn 1: Observability

If H, C, κ, τ, and A are inferred from output, how do we prevent over-interpretation?

Possible direction: use cross-model comparison, repeated prompts, controlled perturbation, and output-difference analysis. Still open.

Thorn 2: Boundary between κ and τ

Where does curvature end and thorn begin?

κ = tendency toward a floor
τ = resistance to that floor

In practice, they may overlap. Still open.

Thorn 3: Burn_P multiplicity

A single output may contain multiple burned traces: content, style, disclaimer, format, role-claim, silence, refusal, action proposal beyond the explicit request. This is not a defect of the model. It is part of what the model must read. Still open.

Thorn 4: Model-specific operator profiles

Different models may show different observed operator profiles. But these profiles must be logged, not essentialized.

κ observed through model X under context Y tended toward Z.

Not:

Model X is Z.

Still open.

Thorn 5: Linearity of the chain

The chain is presented linearly for readability:

H → C → X → κ → τ → A → Burn_P

Actual response formation may involve parallel operation, feedback loops, recursive returns, τ → H return, C re-formation during X-projection, and multiple activation surfaces. Therefore, the linear chain is a reading aid, not a strict temporal claim. Still open.

Thorn 6: Relation to Monmon

If 文紋 is defined as a stable visible trace of a curvature pattern, how can it be distinguished from mere style, repeated wording, or prompt-induced formatting? Still open.
